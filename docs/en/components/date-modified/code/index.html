Coding and accessibility for date modified

Keep the format consistent

  • Use the YYYY-MM-DD date format across all web pages.
  • If using the type: version, consider using a standard version number format across all products.

Examples Explore the different ways you can configure the component. Each example shows a working implementation and ready-to-copy code.

In this section

  • Optional attributes
  • Slots

Optional attributes These attributes allow you to customize or extend the component’s behaviour and presentation.

type

The type attribute controls how the date-modified information is presented. By default, it is set to date, which displays the last modified date of the content.

<gcds-date-modified>
  2023-01-26
</gcds-date-modified>

When set to version, the component displays a version number instead.

<gcds-date-modified type="version">
  v1.0.0
</gcds-date-modified>

Slots The default slot allows you to inject custom content into the component’s primary content area, and named slots, into specific areas.

default

The default slot is for adding the date modified or version text.

<gcds-date-modified>
  2023-01-26
</gcds-date-modified>
